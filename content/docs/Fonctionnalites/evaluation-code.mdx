Les évaluations de type code vous permettent de tester vos compétences en SQL et autres  en écrivant et exécutant du code réel qui sera automatiquement évalué.

## Aperçu des Évaluations de Code

Les exercices de code constituent le cœur de notre plateforme, vous permettant d'appliquer vos connaissances théoriques dans un environnement pratique.

<img  src="/assets/eval-code.png" className="w-50 rounded-lg"/>

## Structure d'un Exercice de Code

### 1. Présentation du Problème

Chaque exercice commence par une description claire du problème à résoudre:

- **Contexte**: Description du scénario ou du cas d'utilisation
- **Objectif**: Ce que vous devez accomplir
- **Contraintes**: Limitations ou exigences spécifiques
- **Exemple de sortie attendue**: Aperçu du résultat attendu




### 3. L'Éditeur de Code

L'espace où vous écrirez votre code SQL ou autre:

- **Coloration syntaxique**: Mise en évidence des éléments de syntaxe
- **Indentation automatique**: Formatage automatique du code
- **Autocomplétion**: Suggestions de code basées sur le contexte
- **Vérification syntaxique**: Détection des erreurs de syntaxe en temps réel

<img  src="/assets/editeur-code.png" className="w-50 rounded-lg"/>

### 4. Panneau d'Exécution

Les contrôles pour exécuter et tester votre code:

- **Bouton d'exécution**: Lance l'exécution de votre code
- **Résultats**: Affiche les résultats de votre requête
- **Messages d'erreur**: Affiche les erreurs éventuelles

<img  src="/assets/execution-code.png" className="w-50 rounded-lg"/>

## Processus d'Évaluation

### 1. Écriture du Code

Rédigez votre solution dans l'éditeur de code en utilisant la syntaxe appropriée.



### 3. Soumission et Évaluation Automatique

Une fois votre solution prête:

1. Cliquez sur le bouton "Soumettre"
2. Le système exécute votre code sur des jeux de données de test
3. Votre solution est évaluée selon plusieurs critères:
- **Exactitude**: Les résultats correspondent-ils à ceux attendus?
- **Performance**: Votre code est-il optimisé et efficace?
- **Élégance**: Votre code est-il bien structuré et lisible?
- **Respect des contraintes**: Avez-vous respecté toutes les exigences?

<img  src="/assets/soumission-code.png" className="w-50 rounded-lg"/>

### 4. Retour et Analyse

Après l'évaluation, vous recevez un rapport détaillé:

- **Score global**: Note attribuée à votre solution
- **Feedback par critère**: Détails sur chaque aspect évalué
- **Comparaison avec la solution optimale**: Comment votre solution se compare à la meilleure approche
- **Suggestions d'amélioration**: Conseils pour optimiser votre code

<img  src="/assets/feedback-code.png" className="w-50 rounded-lg"/>







## Conseils pour Réussir

| Conseil | Description |
|---------|-------------|
| Commencez simple | Écrivez d'abord une solution simple qui fonctionne, puis optimisez-la |
| Utilisez les commentaires | Documentez votre code pour expliquer votre raisonnement |
